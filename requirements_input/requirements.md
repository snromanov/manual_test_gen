## REQ_001 [Front] Отображение календаря событий
- Пользователь открывает раздел календаря и видит список/сетку событий.
- Для каждого события отображаются основные атрибуты (название, дата, статус).
- Интерфейс корректно работает при отсутствии событий.

## REQ_002 [Front] Создание и редактирование события
- Пользователь может создать событие с обязательными полями.
- При сохранении выполняется валидация обязательных полей.
- Пользователь может редактировать существующее событие.

## REQ_003 [Back] Управление статусами события
- Событие поддерживает жизненный цикл со статусами (например, черновик/запланировано/завершено).
- Переходы между статусами валидируются по правилам системы.
- Некорректный переход статуса блокируется с сообщением об ошибке.

## REQ_004 [Front] Фильтрация и поиск событий
- Пользователь может фильтровать события по периоду и статусу.
- Пользователь может искать событие по названию.
- При комбинированных фильтрах выводятся только релевантные результаты.

## REQ_005 [Back] Валидация входных данных и обработка ошибок
- Система отклоняет некорректные форматы данных (дата, длина полей, спецсимволы).
- При ошибках валидации пользователь получает понятное сообщение.
- Ошибочные операции не приводят к повреждению данных.
